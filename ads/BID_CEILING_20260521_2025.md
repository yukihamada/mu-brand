## なぜこれを打つのか

`bump_max_cpc.py` (commit 23f9553) で MU-CRAFT (MANUAL_CPC) には ¥250 入った
が、 同 md §5 のとおり 残り 4 ENABLED は smart-bid (ad_group.cpc_bid_micros
無視) のため別経路必要。 内訳:

- **TARGET_SPEND** 3 件 → `bidding_strategy.target_spend.cpc_bid_ceiling_micros`
  を campaign 単位で SET すれば Google bidder が auction 底値まで張れる
- **MAXIMIZE_CONVERSIONS** (MU-PMax) → 別 agent 担当、本 script では触らない

このタスクは TARGET_SPEND 3 件の ceiling SET だけを扱う。

## Scope (実際に触ったもの)

- `campaign.target_spend.cpc_bid_ceiling_micros` = 250_000_000 (¥250)
  を update_mask 限定で SET
- 対象: ENABLED かつ `bidding_strategy_type == TARGET_SPEND` かつ
  portfolio strategy 未使用 (Standard TARGET_SPEND のみ)

## NOT touched (意図的)

- daily budget (各 campaign の `campaign_budget.amount_micros`)
- campaign status (ENABLED / PAUSED そのまま)
- bidding strategy (TARGET_SPEND を別戦略に変更しない)
- MU-PMax (MAXIMIZE_CONVERSIONS) — 別 agent 担当
- MU-CRAFT-OneClick-2026-05 (MANUAL_CPC) — 前 work `23f9553` で完了済み
- stray `Campaign #1` (PAUSED) — 完全に除外
- ad copy / ad group 構造 / keyword / negative keyword
- `~/.config/google-ads/google-ads.yaml` の値は出力に echo しない

## 安全 gate

- ENABLED daily budget 合計: **¥6,000/day** (abort threshold ¥10,000/day を
  40% 下回るので通過)
- 合計が threshold 超なら exit 1 + 警告 (prompt なし)
- 既に ceiling=¥250 なら skip (idempotent、 再実行可能)
- update_mask は `protobuf_helpers.field_mask` で生成し
  `target_spend.cpc_bid_ceiling_micros` のみ送信 (他 field 巻き込みなし)
