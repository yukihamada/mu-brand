## Scope (実際に触ったもの)

- `ad_group.cpc_bid_micros` を 250_000_000 (¥250) に SET — 対象は ENABLED かつ
  `bidding_strategy_type ∈ {MANUAL_CPC, MANUAL_CPC_ENHANCED}` の campaign のみ
- `ad_group_criterion.cpc_bid_micros` の explicit override (> 0 かつ < target) を ¥250 に SET
  - 0 (= ad_group default を継承) は触らない。継承を override に変換しない

## NOT touched (意図的)

- daily budget (各 campaign の `campaign_budget.amount_micros`)
- campaign status (ENABLED / PAUSED そのまま)
- bidding strategy (TARGET_SPEND / MAXIMIZE_CONVERSIONS を MANUAL_CPC に強制変換しない)
- stray `Campaign #1` (PAUSED) は完全に除外
- ad copy / ad group 構造 / negative keyword

## 安全 gate

- ENABLED campaign の daily budget 合計を実行前に算出:
  **¥6,000/day** (abort threshold ¥10,000/day を 40% 下回るので通過)
- 合計が threshold 超なら exit 1 + 警告 (prompt なし)
- 既に ¥250 なら skip (idempotent、 再実行可能)

## 5. 重要な発見 (今回適用できなかった 4 campaign について)

**5 件 ENABLED のうち 4 件 (MU-AdsTees-Search / MU-Brand / MU-Discovery / MU-PMax) は
`ad_group.cpc_bid_micros` を SET しても効かない。** 理由:

- **TARGET_SPEND** = Maximize Clicks 戦略。Google が自動で入札額を決める。
  ad_group.cpc_bid_micros は ignored。 ceiling を設けたい場合は
  `bidding_strategy.target_spend.cpc_bid_ceiling_micros` を campaign 単位で設定する必要あり。
- **MAXIMIZE_CONVERSIONS** (PMax): cpc 自体が auction 単位で Google 決定。
  Manual で ceiling を引き上げる API がそもそも存在しない (conversion 学習に委ねる設計)。

これらを ¥250 に揃えたい場合の選択肢 (今回のタスク範囲外):

1. **TARGET_SPEND 3 件** → `cpc_bid_ceiling_micros` を 250_000_000 に SET
   (新規 script。 ユーザー承認 + draft md が必要)
2. **TARGET_SPEND → MANUAL_CPC への切替** → 配信パターンが激変するので避ける推奨
3. **MU-PMax** → 触らない (PMax は学習任せが鉄則)

draft md の "max_cpc ¥80-120 が auction 底値を割っている" 仮説 H1 については:
- MU-CRAFT-OneClick-2026-05 → 今回 ¥250 に bump 済み、impr 計測待ち
- TARGET_SPEND 3 件 → 自動入札なので Google が auction 価格を見て自動調整しているはず。
  それでも impr=0 なら根本原因は bid ではなく品質スコア / landing page / KW 関連性。
  別 work で `search_term_view` + `keyword_view` の impression_share を取得して再診断推奨。
