## 推奨セクション (Yuki が書く骨組み)

以下は提案であり、Yuki は自由に削除・追加・改変できる。

### 1. 宛先 / Salutation

> 例: 「これを読む人へ」「次代の Stewards へ」「未来の MU 信者へ」

誰に向けた最後の言葉か。複数宛先でも良い。

### 2. 死を語る / On Death

死の事実をどう受け止めてほしいか。仏教的「無」、ストア哲学的「同意」、柔術家らしい「次のラウンド」、何でも良い。MU の哲学と死の哲学を接続する数行。

### 3. MU の未来へ / To MU's Future

死亡時点で MU が達成していること・未達成のこと。
- 達成済みの誇り
- 未達成への謝罪 (もしあれば)
- 「これだけはやり遂げてくれ」という遺命 (1-3 個に絞る)

### 4. 7 原則の解釈権の譲渡 / Transfer of Interpretive Authority

第三条「7 原則」の解釈に迷ったとき、誰に従うべきか。
- 「現任 Stewards 多数決に従え」
- 「迷ったら一. (摩擦を減らせ) を優先せよ」
- 「私の生前の発言録を参照せよ」

### 5. 個人的なお礼 / Personal Thanks

固有名詞で感謝を伝える人々。家族、共同創業者、初期支援者、Founder Relay の連鎖メンバー、特に世話になった人。
- 名前と感謝の一言

(注: PII 配慮 — 個人 email/住所は書かない、名前のみ)

### 6. 後悔と謝罪 / Regrets and Apologies

もしあれば。なくても良い。Yuki が「あれは間違いだった」と認めたい判断、傷つけた人がいれば。これは強さを増す。

### 7. 最後の願い / Final Wish

葬儀の形、遺骨の処理、遺品の処理、家族への要望。葬儀規約 (06) に法的詳細はあるが、個人的願いはここに書く。

### 8. 締めの一言 / Closing

quotable な 1-2 行。Yuki らしい締めくくり。墓碑銘になり得る言葉。

### 9. 署名 / Signature

```
濱田 優貴
[執筆日]
[執筆場所]
```

## 形式

- **長さ**: 制限なし。Buffett の手紙くらいでも 7 行でも良い。
- **言語**: JA 主、EN 翻訳は任意 (死後に AI 翻訳でも可)
- **媒体**: 紙 (手書き or 印刷) を正本とし、封筒に封印
- **電子コピー**: 暗号化 PDF を 3 ヶ所に保管 (Atelier 金庫、東京拠点金庫、海外 backup)
- **複製禁止**: Yuki 生存中、本人以外の閲覧禁止

---

## 封印手順

1. Yuki が完成稿を紙に印刷 (or 手書き)
2. 封筒に入れる
3. 封筒口に Yuki 署名 + 蜜蝋封印 + 日付
4. 暗号化 PDF を生成 (PGP encrypted to Stewards 全員 + Foundation 理事 multi-sig 必要)
5. PDF を 3 ヶ所に保管
6. bim.house succession_token に「Final Letter Sealed YYYY-MM-DD」を記録 (本文は記録しない)
7. ポンの電子契約に「Final Letter 封印完了」を Yuki 単独署名で確定

## 改訂

Yuki は生存中、何度でも本 Letter を書き換えてよい。改訂時は:

1. 旧封印を Yuki 立会のもと開封 (公衆は立会わない、Yuki 1 人で可)
2. 新版を書く
3. 上記の封印手順を再実行
4. bim.house succession_token に「Revised YYYY-MM-DD」を append

最終バージョンが死亡時点の決定稿となる。
