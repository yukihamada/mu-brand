## 0. 設計原則 — なぜ「無人」が「安全」で「大規模」なのか

普通のフェスは「人手 = 規模 = リスク」。スタッフを増やすほど大きくでき、増やすほど事故も増える。
無音祭はこれを逆転させる：

- **サイレントディスコ（全員ヘッドホン）** → スピーカー無し = 近隣騒音ゼロ = 会場許可が軽い、音響クルー不要、PA事故ゼロ。
- **音源がデジタル同期** → ステージ設営・音響オペレータが要らない。スケールは「サーバーの同時接続数」だけが上限 = 物理的に青天井。
- **入場 = MUGEN の T シャツ** → 物販ゲート・チケットもぎり・現金が要らない。着ているか否かだけ。
- **運営が自律エージェント** → 当日の判断の大半（音量・人流・天候・決済）を機械が回し、人間は止めるボタンだけ持つ。

結論：**無人化はコスト削減ではなく、安全装置であり、スケール装置である。**

## 2. 7つの企画 × 「最小・最大・安全」での実現方法

### ① AI DJ「無人の選曲」
- **最小**: DJ 0人。中村兄弟の音源資産 + Suno 生成セットを、当日リアルタイムでAIが編成。
- **最大**: 1つの音源を全端末に配信するだけなので人数無制限。
- **安全**: 事前に「承認済みプレイリスト」だけを再生（生成物は前日までに人間が一括承認＝著作権・不適切曲の事故防止）。当日AIは"並べ替え"のみで、新規生成はしない。フォールバック=固定セットが常に裏で回る。

### ② 群衆の心拍が BPM になる
- **最小**: 入口で任意計測（スマホのカメラ/つながりAI技術）。スタッフ不要、本人操作。
- **最大**: 平均値だけ使うので母数が増えるほど安定。
- **安全**: 心拍データは**その場で平均化して即破棄**（個人を保存しない＝PII事故ゼロ）。BPMには上下限ガード（例 60–140）。異常値は無視。

### ③ ステージが無い（全端末同期の一斉発音）
- **最小**: 物理ステージ・照明卓ゼロ。Web Audio + サーバー時刻同期。
- **最大**: CDN配信＋クライアント側再生なのでサーバー負荷は軽い。
- **安全**: 同期の"一斉発音"は**音量上限をクライアント側でハードキャップ**（端末スピーカー/ヘッドホン難聴防止）。失敗端末は自動で無音にフォールバック。

### ④ サンセット = 唯一の"音"（生音解禁の数分）
- **最小**: 日没時刻は天文計算で自動。合図も自動。
- **最大**: 「一日で一番静かに感じる一番大きな瞬間」= 体験のクライマックス、SNS拡散の核。
- **安全**: 生音解禁は**音量・時間を事前確定**し、現地のデシベル計と連動。超えたら自動で戻す。ここを**号外Tの最終解禁＝座標公開**に重ねる（§後述）。

### ⑤ 柔術ロール in サイレンス
- **最小**: マット1枚 + 有段者の自主進行。あなたの世界の人脈で回る。
- **最大**: 観るのは無制限、組むのは予約制（安全のため人数制限）。
- **安全**: **ここだけは人間の有資格者（レフェリー/インストラクター）必須**。怪我リスクがある唯一の枠なので、自動化しない＝「機械がやらない判断」の明示。保険・同意書・マット安全を人間が担保。

### ⑥ MUGEN T = あなたのチャンネル
- **最小**: T のシリアル/QR が個人プレイリストの鍵。配布作業ゼロ（既にQR実装あり）。
- **最大**: 着ている全員が自分のchを持てる。号外コレクション数で聴けるch数が増える（既存の報酬ラダーと接続）。
- **安全**: 認証は読み取り専用。個人情報は持たず、シリアル→ch のマッピングだけ。

### ⑦ Koe で"声だけの出演者"
- **最小**: Koeデバイス設置のみ。出演者の渡航・ギャラ・楽屋ゼロ。
- **最大**: 観客の声がその場で環境音に溶ける = 全員が匿名の出演者。
- **安全**: 声は**加工して匿名化、保存しない**。不適切音声はAIフィルタ＋人間の緊急ミュート。

## 4. 絶対安全の仕組み（フェイルセーフ設計）

「自動 = 暴走したら怖い」を、**多層の止め方**で潰す：

1. **物理キルスイッチ** — オペレーターの手元に「全音停止」1ボタン。押すと全端末が即無音＋避難アナウンス（多言語・事前録音）。
2. **デッドマン監視** — 監視が一定時間応答しないと、システムが自動で安全モード（音量を下げ、人間に通知）。MU watchdog の思想。
3. **段階権限** — 自動レイヤーは"可逆な操作"しかできない。不可逆（座標公開・返金確定・避難）は人間ゲート必須。誤クリック防止の二段確認。
4. **天候フェイルセーフ** — 気象APIと連動。基準超で自動的にプログラム短縮/中止提案を人間に上げる。
5. **決済の分離** — 当日販売は最小限。MUGEN T は事前にオンラインで売り切る設計（現地で現金を扱わない＝盗難・釣り銭事故ゼロ）。
6. **データ最小主義** — 心拍・声・位置は**その場で集計して破棄**。保存しない＝漏洩しようがない。これがブランドの信頼そのもの。
7. **オフライン耐性** — 会場の回線が落ちても、各端末にプリロードした"裏セット"が鳴り続ける。サーバー依存で全停止しない。

## 5. 最大規模の仕組み（スケールの上限を外す）

- **音はクライアント再生** → 同時1万人でもサーバーは「時刻合わせ」を配るだけ。重い処理が無い。
- **入場はTシャツ** → ゲートの処理速度が律速にならない。人が流れる。
- **遠隔参加** → 会場は10/29ハワイの一日だが、号外Tは世界中で買える＝"着ること自体が参加"。物理会場の収容を超えて「参加者数」を伸ばせる。
- **メディアが勝手に増幅** → 「世界一静かな音楽フェス」「無音の群衆」は撮りたくなる絵。広告費ゼロで拡散する設計（既存のシェアキット＋OGP）。

## 6. 号外Tとの接続（既に動いている部分）

- 開催まで毎日「号外」T を MUGEN で販売（¥22,500 限定／ドラフト→承認制）。
- **共鳴カウンター**: 号外の累計販売が 108 に届くと会場ヒント→**座標解禁**。これを当日のサンセット生音解禁（企画④）に重ねる＝オンラインとオフラインのクライマックスを同期。
- **+1招待ツリー**: MUGENを着た者が無音の者を1人連れて入れる＝来場が物理的に倍化。
- **報酬ラダー**: 号外コレクション数 → 当日の聴けるチャンネル数（企画⑥）。

→ つまり**集客（オンライン）と体験（オフライン）が一本の仕掛けで繋がっている**。
