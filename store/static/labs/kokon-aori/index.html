02 ナレーション

ElevenLabs v3 / 男性低音・武骨 (Otani voice / 3JDquces8E8bkmvbh6Bc)。

原稿

今夜、東京で、火が点く。

保険会社・フレックスグループ。 焼肉屋・古今。

数字で守る者と、火で語る者。

五対五。 逃げ場の無い、マットの上。

一人目、角度で「きめる」。 二人目、重量で攻める。
三人目、失うものが、無い。 四人目、二十年磨いた職人。
五人目、肉も「きめ」も、火加減一つ。大将。

勝てば、古今で祝勝会。 負ければ、古今で反省会。

…どっちにしろ、古今。

観戦の余韻は、炭の上で。
読み方ハック:「極める」は v3 がデフォルトで「きわめる」と読む。 柔術用語は「きめる」が正しい。「きめる」 と仮名で囲って書くと v3 がそのまま読んでくれる。 narration locked 後に直すと音声差し替えになるので、最初から仮名指定がおすすめ。

呼び出し例 (Python)

import os, json, urllib.request, base64
text = open('narration.txt').read().strip()
voice_id = "3JDquces8E8bkmvbh6Bc"  # Otani
url = f"https://api.elevenlabs.io/v1/text-to-speech/{voice_id}/with-timestamps?output_format=mp3_44100_192"
payload = {
    "text": text,
    "model_id": "eleven_v3",
    "voice_settings": {
        "stability": 0.45, "similarity_boost": 0.8,
        "style": 0.55, "use_speaker_boost": True
    }
}
req = urllib.request.Request(url, data=json.dumps(payload).encode(),
    headers={"xi-api-key": os.environ['ELEVENLABS_API_KEY'],
             "Content-Type": "application/json"})
with urllib.request.urlopen(req, timeout=180) as r:
    data = json.loads(r.read().decode())
open('narration.mp3','wb').write(base64.b64decode(data['audio_base64']))
# alignment ← 字幕生成 (後述) に使う
open('alignment.json','w').write(json.dumps({
    'characters': data['alignment']['characters'],
    'starts':     data['alignment']['character_start_times_seconds'],
    'ends':       data['alignment']['character_end_times_seconds'],
}, ensure_ascii=False))

with-timestamps エンドポイントを使うと、文字ごとの開始/終了秒 が alignment に入って返ってくる。これが 字幕の自動生成 (Step 09) にそのまま使える。

04 BGM (ElevenLabs Music)

POST https://api.elevenlabs.io/v1/music/compose
{
  "prompt": "Cinematic Japanese fight-night trailer score: deep sub bass drone,
             ascending strings, occasional taiko hits, slow build into massive impact
             around 50 seconds, then warm release with low chord. Dark, intense,
             no vocals, no lyrics, no melody-driven motif.",
  "music_length_ms": 67000
}

無メロディ・無歌詞 を明示しないと EDM 的なメロディが出てくる。 「no vocals, no lyrics, no melody-driven motif」をプロンプトに必ず入れる。

ナレーション・boom・tom と被るので、最終的に BGM は volume=0.32 で バックに敷くだけにする。BGM を主役にすると言葉が聴こえなくなる。

06 Veo 3.1 i2v — マスコット動かし

kokon.tokyo の公式マスコット「炎くん」(honoo) の静止 PNG を image-to-video で アニメーション化。背景固定が肝。

ハマりポイント: 普通のプロンプトだと Veo は途中で背景を黒→白に 切り替えてくる。後段の colorkey で抜けなくなって背景が白く残る。 プロンプトに DARK SOLID BLACK background, pure RGB (0,0,0) for entire 8 seconds, no background change, no flash と明示。さらに negativePrompt に white background, light background, color change, flash。

呼び出し (image-to-video)

import base64, json, urllib.request, os
API = os.environ['GEMINI_API_KEY']
URL = f"https://generativelanguage.googleapis.com/v1beta/models/veo-3.1-fast-generate-preview:predictLongRunning?key={API}"

DARK = ("DARK SOLID BLACK background, pure RGB (0,0,0) for entire 8 seconds, "
        "no background change, no flash, no white, no color shift, "
        "only the character animates. ")

img_b64 = base64.b64encode(open("honoo_happy.png","rb").read()).decode()
body = json.dumps({
    "instances": [{
        "prompt": DARK + "Cute mascot flame character with kanji '古今' on apron, "
                  "jumping in celebration with arms raised, sparkles fly around, "
                  "looping joyful dance, 2D illustrated style.",
        "image": {"bytesBase64Encoded": img_b64, "mimeType": "image/png"}
    }],
    "parameters": {
        "aspectRatio": "16:9", "durationSeconds": 8,
        "personGeneration": "allow_all",
        "negativePrompt": "white background, light background, color change, flash, "
                          "text, watermark, logo, ui overlay, music, dialogue"
    }
}).encode()
op = json.loads(urllib.request.urlopen(urllib.request.Request(URL, data=body,
        headers={"Content-Type":"application/json"})).read())["name"]
# ↑ op を返す。:predictLongRunning は polling 必要。

def slide_fighter(path, num, catch, name, title):

    sx, sy = 1190, 870
    d.rectangle([sx-30, sy-20, 1900, sy+170], fill=(0,0,0,210))
    d.line([sx-30, sy-20, sx-30, sy+170], fill=(200,16,46), width=6)
    d.text((sx, sy),    name,  font=font(FONT_HEAVY, 64), fill=(245,240,235))
    d.text((sx, sy+80), title, font=font(FONT_MID, 36),   fill=(220,200,160))
    img.save(path)

ヒーロー写真背景スライド (顔切れ対策)

ポートレート写真を 1920×1080 横長にカバーフィットすると顔が見切れる。 y_bias を 0.05 (top-aligned) にすると 頭から胸まで確実に残る。

def photo_backdrop(path, dim=0.55, blur=2, saturate=0.85, y_bias=0.5):
    img = Image.open(path).convert("RGB")
    ratio = max(1920 / img.width, 1080 / img.height)
    nw, nh = int(img.width * ratio), int(img.height * ratio)
    img = img.resize((nw, nh), Image.LANCZOS)
    x_off = (nw - 1920) // 2
    y_off = int((nh - 1080) * y_bias)   # 0.0 = top, 0.5 = center, 1.0 = bottom
    img = img.crop((x_off, y_off, x_off + 1920, y_off + 1080))
    if blur:     img = img.filter(ImageFilter.GaussianBlur(radius=blur))
    img = ImageEnhance.Color(img).enhance(saturate)
    img = ImageEnhance.Brightness(img).enhance(dim)
    return img.convert("RGBA")

映像 overlay chain (15 スライド + 5 Veo)

各スライドは fade=t=in / fade=t=out の alpha フェード付きで 透過状態で base に上乗せ。enable='between(t,ST,EN)' で時間窓を切る。

# スライド 1 つ分の処理
[1:v]format=yuva420p,
     scale=w='iw*(1.00+0.035*(max(0,min(3.4,t-61.6))/3.4))':h=-1:eval=frame,  # kenburns
     fade=t=in:st=61.6:d=0.3:alpha=1,
     fade=t=out:st=64.65:d=0.35:alpha=1,
     setpts=PTS-STARTPTS[s14]

# overlay
[prev][s14]overlay=x='(W-w)/2':y='(H-h)/2':
            enable='between(t,61.6,65.0)':
            shortest=0:eof_action=pass[v14]
重要: scale=...:eval=frame の t は video の global 時刻 を指す。そのままだと kenburns が暴走する。 max(0, min(dur, t - slide_start)) でローカル時刻に clip する。

09 アルファ除去 (Veo マスコット)

Veo の i2v 出力には letter-box gray bar が入る。 colorkey black だけだとグレーが残るので、lumakey も チェーンして 暗い灰色も アルファ 0 にする。

[N:v]scale=-2:600,setsar=1,
     colorkey=color=black:similarity=0.30:blend=0.12,   # 真っ黒 → 透過 (強め)
     lumakey=threshold=0.08:tolerance=0.05,             # 暗灰色も 透過
     fade=t=in:st=ST:d=0.3:alpha=1,
     fade=t=out:st=EN-0.3:d=0.3:alpha=1,
     format=yuva420p[honoo]

ベスト プラクティス まとめ

  1. ナレーションは仮名指定で読み方ハック — 専門用語は最初から仮名で書く
  2. Veo の背景は明示的に固定 — i2v だと途中で勝手に背景変わるので prompt で固定
  3. colorkey + lumakey の 2段 — 真っ黒だけだと letterbox 灰色が残る
  4. kenburns の eval=frame はローカル時刻に clip — global t そのままだと暴走
  5. portrait 写真は y_bias=0.05 — center crop だと顔切れる
  6. 字幕は fighter intro 飛ばす — lower-third stats card と被る
  7. BGM は -10dB 以下 — narration の邪魔しない (volume=0.32 推奨)
  8. Veo の foley も低音量で残す — 完全 mute より臨場感↑ (volume=0.07〜0.15)
  9. ハードカット + 白フラッシュ + bigboom を同タイミング — クライマックスを 3 重で殴る
  10. 17 回作り直してもコスト ¥2,000 以下 — 完璧主義で OK
