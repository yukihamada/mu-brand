# §28 MU 利益分配スキーム — 完全公開版

> Constitution §28。 §27 (50% 寄付) を **6 分割スキーム** に拡張・統合した正式版。
> 株式会社イネブラ (Enabler Inc., 本店 東京都千代田区九段南 1-5-6 りそな九段ビル 5F) の
> MU 事業 (wearmu.com / mu-brand) に対して適用。

---

## 1. 概要 (Summary)

MU 事業の **税引後 当期純利益 P** を、 各事業年度の期末監査確定後に、
以下の **6 セグメントに分配する** (合計 100%、 端数 ¥1 まで明示)。

| # | セグメント | 比率 | 受益主体 | 法的性格 |
|---|---|---:|---|---|
| 1 | **寄付** | 50% | 北海道川上郡弟子屈町 (企業版ふるさと納税) | 損金算入 + 法人税特別控除 (実質負担 ~10%) |
| 2 | **Yuki 報酬** | 10% | 濱田優貴 (代表取締役 兼 ディレクター) | 役員報酬 (定期同額給与) として翌期 12 等分 |
| 3 | **株主配当** | 10% | Enabler 株主全員 (持分比率) | 株主総会決議後の配当 (East Ventures 5% 含む) |
| 4 | **MA ホルダー還元** | 10% | MUGEN + stack ホルダー | MU クーポン発行 (前払式支払手段 自家型) |
| 5 | **MU Community Fund** | 10% | コミュニティ (公募 grant) | Enabler 内 引当金 → 50% が community grant 枠 |
| 6 | **運転備金** | 10% | Enabler Inc. 内部留保 | 利益剰余金 (翌期投資・損失バッファ・配当余力) |
| **合計** | | **100%** | | |

P = MU 事業の **税引後** 当期純利益 (法人税・住民税・事業税 控除後)。

---

## 2. なぜこの設計か (Rationale)

### 2.1 50% 寄付 (§27 継承)
MU brand の根本コミットメント。 §27 で「post-tax net の 50% を 弟子屈町へ」と
規定済み。 これを §28 でも継承し、 **企業版ふるさと納税** (内閣府 地方創生応援税制)
を活用することで、 法人としても税制上最も効率的に社会還元できる。

- 弟子屈町は内閣府認定済の対象自治体 ([memory: teshikaga_corporate_furusato](teshikaga_corporate_furusato))
- 寄付金額の **最大約 9 割** が法人税額から控除 (損金算入分 + 特別控除分の合算)
- 実質的に MU の社会還元コストは寄付額の 1 割程度で、 弟子屈町には満額が届く

### 2.2 10% Yuki (役員報酬 / 業務委託の二択)
利益連動 役員報酬は同族会社では損金算入されないため、 **「P × 10%」 をそのまま
業績連動賞与にできない**。 対応:

- **本則**: 期初に予測 P に基づき 翌期 12 等分の定期同額給与 (役員報酬) を設定
- **補正**: 期末監査で確定した P × 10% との差分は、 翌々期の 役員報酬で吸収
- **代替案**: yuki 個人事業主 (mu-brand コンサルティング) への業務委託費として
  月次 invoice 払い (損金算入可、 yuki 側は事業所得申告)

### 2.3 10% 株主配当 (Enabler 全株主)
Enabler Inc. の全株主に持分比率で配当。

- East Ventures 5% (優先株、 SOLUNA 経由出資 [memory: soluna_tapkop](soluna_tapkop))
- 残り 95% は yuki ほか普通株主
- **配当原資**: 当期純利益から株主資本コスト相当を還元
- **手続**: 期末監査済 PL → 定時株主総会で剰余金処分決議 → 振込
- **会計**: 法人税後利益からの分配 (損金不算入)、 株主側で配当控除/源泉徴収

### 2.4 10% MA ホルダー還元 (MUクーポン)
MUGEN を 1 着以上保有し、 かつ他 MU NFT (Constitution / MUON / MA) を保有する
ホルダー (= 「MUGEN+stack」) への還元。

- **NG**: 「NFT を持っていれば JPY 自動分配」 → 金融商品取引法上の有価証券扱いリスク
- **採用**: ホルダーリストに対し **MU クーポン (前払式支払手段、 自家型)** を JPY
  同等価値で発行
- 自家型 = MU 自社サイト (wearmu.com / mu-brand) でのみ使用可能
- 資金決済法 第3条 第4項: 発行残高 ¥1,000万 超時に届出義務、 未使用残高の 50% を
  供託 → 当面は閾値未満で運用、 超えたら関東財務局へ届出
- **会計**: 発行時 引当金 (将来の販売値引き相当)、 利用時 売上値引きとして控除

### 2.5 10% MU Community Fund
コミュニティ (作家招聘、 コラボ予算、 community event) のための **Enabler 内 引当金**。

- **当面トークン化しない** (改正資金決済法 + 暗号資産税制の不確実性回避)
- 内部引当金として JPY のまま保持
- 半期ごとに 50% を community grant 枠として公募支出
  - 残り 50% は将来のトークン化原資、 または翌期 community fund 繰越
- **将来オプション**: community 規模が拡大したら、 関東財務局 暗号資産交換業
  登録済の第三者と協業して **MU Token** を発行検討 (前払式 or セキュリティトークン)
- **NG**: yuki 個人や Enabler が独自に暗号資産を新規発行・配布する → 規制違反リスク

### 2.6 10% 運転備金 (Operating Reserve)
利益剰余金として Enabler Inc. 内部に留保。 在庫リスク、 キャッシュフロー、
想定外の品質クレーム、 来期の生産パートナー前払金、 翌期配当余力に充当。

## 3. 計算式 (Computation)

### 3.1 期末
```
税引前利益 (PBT)  =  売上 − 売上原価 − 販管費
法人税等 (T)      =  PBT × 実効税率 (約 30%)
税引後利益 (P)    =  PBT − T

分配:
  donation_jpy        = floor(P × 0.50)
  yuki_jpy            = floor(P × 0.10)
  shareholder_jpy     = floor(P × 0.10)
  ma_holder_jpy       = floor(P × 0.10)
  community_jpy       = floor(P × 0.10)
  reserve_jpy         = P − 上記 5 項目の合計   # 端数を 運転備金に寄せる
```

端数は **運転備金に寄せる** (内部留保なので最も振れに寛容)。
全 6 項目は JPY 整数として記録 (¥1 単位)。

### 3.2 中間期 (推定値、 /api/transparency 用)
監査確定前は **推定 P** を以下で計算:
```
estimated_PBT = MA 落札額 + 物販粗利 (Printful margin) + 法人粗利 (商業ライセンス)
              − 推定販管費 (固定 ¥X/月)
estimated_T   = estimated_PBT × 0.30
estimated_P   = estimated_PBT − estimated_T
estimated_distribution = estimated_P × 各比率
```

中間値はあくまで estimate。 実支払はすべて期末確定値に基づく。

|---|---|---|

## 5. 透明性 (Transparency)

| 公開 URL | 内容 |
|---|---|
| `/profit-split` | この §28 spec の HTML 版 (markdown レンダリング) |
| `/api/profit-split` | スキーマ + 当期推定 + 過去実績の JSON |
| `/api/transparency` | `profit_split` キーで current snapshot を含む |
| `/donations` | §27 の寄付実績 ledger (append-only md) |

### 5.1 ledger テーブル
`profit_split_distributions` (DB) に **実際に支払った** 分のみ記録:
- `id`, `fiscal_year`, `segment` (donation/yuki/shareholder/ma/community/reserve)
- `amount_jpy`, `recipient`, `tx_ref` (振込明細番号 / ふるさと納税申込番号 / MUクーポンID)
- `paid_at` (実支払日), `recorded_at` (記録日), `note`
- 削除不可 (append-only); 修正は 反転エントリ (negative amount) + 新規エントリ

### 5.2 過去未実績 vs 約束
**現状 (2026 fiscal year)**: 推定 P がまだ少額のため、 実支払はゼロまたは寄付の
floor (¥100,000/年 per §27) のみ。 P が成長したら順次 実支払を開始。

## 6. 法的安全性 (Japan Legal Compliance)

### 6.1 適用法令
- **会社法**: 剰余金処分 (株主総会決議)、 配当限度額 (純資産の範囲内)
- **法人税法**: 寄付金損金算入限度額、 役員給与の損金性 (定期同額/事前確定/業績連動)
- **金融商品取引法**: MA 還元を NFT 配当扱いにしない、 MU Token 新規発行は登録業者経由
- **資金決済法 (改正)**: MUクーポン = 前払式支払手段 自家型 (発行残高 ¥1,000万超で届出)
- **暗号資産税制**: 当面 MU Token 発行しない (個人受領は雑所得 累進 最大 55%)
- **企業版ふるさと納税法 (地方創生応援税制)**: 弟子屈町への寄付は最大 9 割税控除

### 6.2 監査
期末監査は税理士法人 (現在: 検討中、 East Ventures 経由で紹介予定) に依頼。
監査結果に基づき 各分配額を確定。

### 6.3 リスク
- **役員報酬利益連動**: 同族会社の業績連動給与は損金不算入 → 業務委託契約で迂回可、
  ただし税務調査で「実質的に給与」と認定されるリスクあり → 監査人と相談
- **MA 還元の NFT 配当判定**: ホルダーに対して継続的・自動的に金銭分配すると
  「集団投資スキーム持分」 (金商法 2 条 2 項) 該当の可能性 → クーポン (財貨提供)
  で回避
- **MUクーポン発行残高**: ¥1,000万超で関東財務局届出 → 監視ロジックを実装
- **配当原資不足**: 会社法 461 条 (配当規制) で純資産 ¥300万未満では配当不可 →
  事前に純資産確認
