呼び出し例 (Python)

import os, json, urllib.request, base64
text = open('narration.txt').read().strip()
voice_id = "3JDquces8E8bkmvbh6Bc"  # Otani
url = f"https://api.elevenlabs.io/v1/text-to-speech/{voice_id}/with-timestamps?output_format=mp3_44100_192"
payload = {
    "text": text,
    "model_id": "eleven_v3",
    "voice_settings": {
        "stability": 0.45, "similarity_boost": 0.8,
        "style": 0.55, "use_speaker_boost": True
    }
}
req = urllib.request.Request(url, data=json.dumps(payload).encode(),
    headers={"xi-api-key": os.environ['ELEVENLABS_API_KEY'],
             "Content-Type": "application/json"})
with urllib.request.urlopen(req, timeout=180) as r:
    data = json.loads(r.read().decode())
open('narration.mp3','wb').write(base64.b64decode(data['audio_base64']))
# alignment ← 字幕生成 (後述) に使う
open('alignment.json','w').write(json.dumps({
    'characters': data['alignment']['characters'],
    'starts':     data['alignment']['character_start_times_seconds'],
    'ends':       data['alignment']['character_end_times_seconds'],
}, ensure_ascii=False))

with-timestamps エンドポイントを使うと、文字ごとの開始/終了秒 が alignment に入って返ってくる。これが 字幕の自動生成 (Step 09) にそのまま使える。

ヒーロー写真背景スライド (顔切れ対策)

ポートレート写真を 1920×1080 横長にカバーフィットすると顔が見切れる。 y_bias を 0.05 (top-aligned) にすると 頭から胸まで確実に残る。

def photo_backdrop(path, dim=0.55, blur=2, saturate=0.85, y_bias=0.5):
    img = Image.open(path).convert("RGB")
    ratio = max(1920 / img.width, 1080 / img.height)
    nw, nh = int(img.width * ratio), int(img.height * ratio)
    img = img.resize((nw, nh), Image.LANCZOS)
    x_off = (nw - 1920) // 2
    y_off = int((nh - 1080) * y_bias)   # 0.0 = top, 0.5 = center, 1.0 = bottom
    img = img.crop((x_off, y_off, x_off + 1920, y_off + 1080))
    if blur:     img = img.filter(ImageFilter.GaussianBlur(radius=blur))
    img = ImageEnhance.Color(img).enhance(saturate)
    img = ImageEnhance.Brightness(img).enhance(dim)
    return img.convert("RGBA")

ベスト プラクティス まとめ

  1. ナレーションは仮名指定で読み方ハック — 専門用語は最初から仮名で書く
  2. Veo の背景は明示的に固定 — i2v だと途中で勝手に背景変わるので prompt で固定
  3. colorkey + lumakey の 2段 — 真っ黒だけだと letterbox 灰色が残る
  4. kenburns の eval=frame はローカル時刻に clip — global t そのままだと暴走
  5. portrait 写真は y_bias=0.05 — center crop だと顔切れる
  6. 字幕は fighter intro 飛ばす — lower-third stats card と被る
  7. BGM は -10dB 以下 — narration の邪魔しない (volume=0.32 推奨)
  8. Veo の foley も低音量で残す — 完全 mute より臨場感↑ (volume=0.07〜0.15)
  9. ハードカット + 白フラッシュ + bigboom を同タイミング — クライマックスを 3 重で殴る

13.3 · そのまま 動く コマンド 一式

(a) ElevenLabs v3 narration

curl -X POST "https://api.elevenlabs.io/v1/text-to-speech/$VOICE_ID" \
  -H "xi-api-key: $ELEVENLABS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "(narration.txt 全文)",
    "model_id": "eleven_v3",
    "voice_settings": {"stability": 0.55, "similarity_boost": 0.75, "style": 0.30}
  }' --output narration_v3.mp3

(b) ElevenLabs Music BGM (climax build)

curl -X POST "https://api.elevenlabs.io/v1/music/compose" \
  -H "xi-api-key: $ELEVENLABS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "dark tribal taiko intro, slow build, climax with synth riser and 808 sub drop at 45s mark, total 60 seconds",
    "music_length_ms": 60000
  }' --output bgm_climax.mp3

(c) Veo 3.1 fast — 炎 mascot i2v

curl -X POST "https://generativelanguage.googleapis.com/v1beta/models/veo-3.1-fast-generate-preview:predictLongRunning" \
  -H "x-goog-api-key: $GEMINI_API_KEY" -H "Content-Type: application/json" \
  -d '{
    "instances": [{
      "prompt": "the flame character meditates, then erupts upward. DARK SOLID BLACK background, pure RGB (0,0,0) for entire 8 seconds, no background change, no flash, no white",
      "image": { "bytesBase64Encoded": "(炎くん PNG base64)" },
      "negativePrompt": "white background, light flash, color shift to white"
    }],
    "parameters": { "aspectRatio": "16:9", "durationSeconds": 8 }
  }'

(d) ffmpeg core composite — dual key で 黒 抜き

ffmpeg -i bg.mp4 -i honoo_meditation.mp4 -filter_complex "
  [1:v]colorkey=color=black:similarity=0.30:blend=0.12,
       lumakey=threshold=0.08:tolerance=0.05,
       scale=1280:720[honoo];
  [0:v][honoo]overlay=(W-w)/2:(H-h)/2:enable='between(t,12,20)'[v]
" -map "[v]" -map 0:a -c:v libx264 -preset slow -crf 18 out.mp4

(e) ken-burns の eval=frame 罠 回避

scale=w='if(eq(on,1),iw,ow)':h='if(eq(on,1),ih,oh)':eval=frame,
zoompan=z='1+0.15*max(0,min(dur,t-slide_start))/dur':
        d=125:s=1920x1080

↑ t を そのまま 渡すと global timeline で 暴走する。 max(0,min(dur,t-slide_start)) で local clip 必須。

(f) m5 Mac SSH ループ (build → 再生 → 修正)

rsync -avz --exclude assets/ ./ yukihamada@192.168.0.5:/tmp/kokon_aori/
ssh yukihamada@192.168.0.5 'cd /tmp/kokon_aori && python3 build_video_v17.py'
scp yukihamada@192.168.0.5:/tmp/kokon_aori/out_v17.mp4 ./
